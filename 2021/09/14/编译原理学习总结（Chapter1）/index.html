第一章：绪论

编译与解释

编译

编译器就是一个程序，它读入用某种语言编写的源程序，并翻译成一个与之等价的另一种语言编写的源程序。常见的编译型语言比如C语言，它在经编译器编译后会生成一个与之等价的exe文件，计算机可以直接执行该文件，此过程类似于英文翻译中的笔译。

解释

把源语言写的源程序作为输入，但不产生目标程序，而是边解释便执行源程序本身。常见的解释型语言比如Python，一个Python程序经Python解释器解释后并不会直接生成一个计算机可执行的程序，而是便执行边对源代码进行解释，此过程类似于英文翻译中的同声传译。

编译过程概述

过程

编译过程主要分为以下几个步骤：

  1. 词法分析
  2. 语法分析
  3. 语义分析
  4. 中间代码生成
  5. 代码优化
  6. 目标代码生成

以 a = b + c * d为例，具体过程如下：

词法分析

  1. 读入源程序
  2. 识别出程序中的各单词：a、=、b、+、c、*、d
  3. 用记号方式表示识别出的单词
  4. 词法检查：检查词法错误，主要是检查单词中使用了非法字符错误
  5. 删除空白符、注释等

语法分析

  1. 在词法分析的基础上，根据语言的语法规则，将单词符号串组成各类语法单位。

具体来说，语法分析是在单词流的基础上建立一个层次结构——建立语法树。

  1. 检查形式语法错误：

例如：Y := A + B; //C语言赋值语句应为 =

​ (x + y)) * z //括号不匹配

语义分析

检查（静态）语义错误，提取语义信息，即利用语法分析阶段确定的层次结构来识别表达式及语句中的操作信息及类型信息。

语义错误分为静态和动态两种：

  1. 静态语义错误

在编译期间可以检查到的语义错误，例如两指针相加或使用未声明的变量这种行为，这类错误如今功能强大的IDE一般能在编写代码的过程中指出。

  1. 动态语义错误

在目标程序运行时才可以检查到的语义错误，例如除数为0等错误。

中间代码生成

在源程序的语法分析和词法分析后，很多编译器会生成一个明确的低级的或类机器语言的中间表示。我们可以把这个表示看作是某个抽象机器的程序。这种中间表示应该具有两个重要性质：

  1. 易于生成
  2. 能够轻松的翻译成目标机器上的语言

常见的中间表示形式比如三地址码（四元式）：

temp1 = c * d (*, c, d, temp1)

代码优化

试图改进中间代码，以便生成更好的目标代码。例如将 a = b + c; d = a; 优化成 d = b + c;

优化的种类包括表达式的优化、公共子表达式的优化、循环优化等等。

目标代码生成

代码生成器以源程序的中间表示形式作为输入，并把它映射到目标语言。如果目标语言是机器代码，那么就必须为程序使用的每个变量选择寄存器或内存位置。然后，中间代码被翻译成为能够完成相同任务的机器指令序列。

目标代码的三种形式：

  1. 绝对指令代码：具有绝对地址的机器指令代码，无需连接即可运行。
  2. 可重定位的指令代码：具有浮动地址的机器指令代码，需连接装配后方能运行。
  3. 汇编指令代码：汇编语言形式的目标程序，需经汇编程序汇编产生相应的机器代码。

编译各阶段的分组

有时可以将编译过程分为前端和后端

前端

  1. 前端包括词法分析、语法分析、语义分析，以及相关的错误处理和符号表的建立。

  2. 前端依赖于源程序且很大程度上独立于目标机器。

  3. 前端生成与平台无关的字节码。

后端

  1. 后端依赖于目标机器。
  2. 后端处理对象是由前端产生的结果，即中间代码。
