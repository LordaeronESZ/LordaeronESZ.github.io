第二章 运算方法和运算器

数据与文字的表示方法

数据格式

计算机中常用的数据表示格式有两种，一种是定点格式，一种是浮点格式。一般来说，定点格式容许的数值范围有限，要求的处理硬件比较简单。而浮点格式容许的数值范围很大，要求的处理硬件比较复杂。

定点数的表示方法

所谓定点数，即小数点位置固定不变。原理上讲，小数点固定在哪一位都可以，但通常将数据表示为纯小数或纯整数。

假设用 n + 1 位字来表示一个定点数 x ， xn 用来表示数的符号，放在最左位置，0 和 1 分别代表正号和负号。

  • 如果表示的是纯小数，那么小数点位于 xn~xn-1 之间，数的表示范围为： 0 <= |x| <= 1 - 2-n.
  • 如果表示的是纯整数，那么小数点位于 x0 的右边，数的表示范围为：0 <= |x| <= 2n - 1.

目前计算机中多采用定点纯整数表示，因此将定点数表示的运算简称为整数运算。

浮点数的表示方法

把一个数的有效数字和数的范围在计算机中的一个存储单元中分别表示，这样的表示方法相当于数的小数点位置随比例因子不同而在一定范围内自由浮动，称之为浮点表示法。

在计算机中任意一个二进制数可以表示成：N = 2e * M.

M称为浮点数的尾数，是一个纯小数。e是比例因子的指数，称之为浮点数的指数，是一个整数。

在计算机表示一个浮点数时，一要给出尾数，用定点小数表示，决定了浮点数的表示精度。二要给出指数，用整数表示，或称阶码，决定了浮点数的表示范围。浮点数同样有符号位。

IEEE754标准下，32 位和 64 位浮点数的标准格式为：

  • S 是浮点数的符号位，S = 0 表示正数，S = 1 表示负数。
  • M是尾数，小数点位置放在尾数域最高有效位的右边。
  • E是阶码，阶码采用隐含方式，即采用移码方法来表示正负指数。采用这种方式时，将浮点数的指数真值 e 变成阶码 E 时，应将指数 e加上一个固定的偏移值 127（01111111），即 E = e + 127。

当尾数的值不为 0 时，尾数域的最高有效位应为 1，这称为浮点数的规格化表示。

在 IEEE754 标准中，一个规格化的 32 浮点数 x 的真值表示为：

x = (-1)s * (1.M) * 2E - 127 e = E - 127

同理，由于 64 位浮点数中尾数域 52 位，指数偏移值为 1023 。因此规格化的 64 位浮点数 x 的真值为

x = (-1)s * (1.M) * 2E - 1023 e = E - 1023

IEEE754 中还有一个代表无效运算的符号，例如 0 / 0 或者无穷减无穷。这个符号是 NaN，表示不是一个数。采用这个标志的目的是让程序员能够推迟进行测试及判断的时间，以便在方便的时候进行。

数的机器码表示

机器码是将符号位和数值位一起编码来表示相应的数的表示方法，有原码、补码、反码、移码。为了区别一般书写表示的数和机器中这些编码表示的数，通常将前者称为真值，后者称为机器码或机器数。

但根据补码的定义可知，计算补码仍然需要进行减法运算，为此可以通过反码来解决。

一个正整数，当用原码、反码和补码表示时，符号位都固定为 0，用二进制表示的数位值都相同，即三种表示完全一致。

一个负整数，当用原码、反码和补码表示时，符号位都固定为 1，用二进制表示的数位值都不相同，满足如下规则：

  1. 原码符号位为 1 不变，整数的每一位二进制数位求反得到反码；
  2. 反码符号位为 1 不变，反码数值位最低位加1， 得到补码。

例如： x = +122，y = -122，求[x]原、[x]反、[x]补、[y]原、[y]反、[y]补。

解：x = (+122)10 = (+1111010)2，y = (-122)10 = (-1111010)2

[x]原 = 01111010，[x]反 = 01111010，[x]补 = 01111010

[y]原 = 11111010，[y]反 = 10000101，[y]补 = 10000110

移码表示法

移码通常用于表示浮点数的阶码。由于阶码是个 k 位整数，假设定点整数移码形式为 ekek-1···e2e1e0 时（最高位为符号位），移码的传统定义为： [e]移 = 2k + e，2k > e >= -2k.

若阶码数值部分为 5 位，以 e 表示真值，则 [e]移 = 25 + e，25 > e >= -25.

例如，当正数 e = +10101时，[e]移 = 110101；当负数 e = -10101时，[e]移 = 25 - 10101 = 001011。最高位为符号位。可见，当 e 为正数时，符号位为 1 ，当 e 为负数时，符号位为 0 。
